git 常用命令行操作

发表于 2017-02-22 | 分类于 learn-git | | 阅读次数

git add .

. 表示将整个working directory都提交到staging area

git diff --cached

show what has been staged and what hasn’t been committed

git rm --cached file_name

把文件从stage中删除，但文件本身不删除

# 添加到stage
git add a.txt

# 只从stage中删除， 保留文件
git rm --cached a.txt

git rm -f file_name

强制删除，不保留文件

# 添加到stage
git add a.txt

# 下面从stage中直接删除不行
git rm a.txt

# 使用强制删除，将文件也删除
git rm -f a.txt

git mv file_a file_b

将 文件a 改名为 文件b

git add a.txt

git commit -m "add file a.txt"

git mv a.txt b.txt

git log

1- git log --pretty=oneline

将日志一行一行的显示出来，只显示SHA和message部分

2- git log --pretty=format:"%h : %an : %ar : %s"

按格式显示log:

  • %h: 显示SHA前7位
  • %an: 显示作者
  • %ar: 显示提交过去了多长时间
  • %s: 显示提交时候的message

3- git log -p -2

显示最后2次的提交

4- git log --since=1.weeks

显示过去1周的提交

5- git log --since="2016-11-08"

还有具体显示某天以后的提交的

6- git log --before="2016-11-27"

显示某天以前的提交

7- git log --author="James Sawyer"

显示该作者的提交

git commit --amend

修改提交的message

git add a.txt

git commit -m 'I just commit it'

git commit --amend

# 然后在terminal中可以修改上面提交的信息

# 修改完成之后按 wq 退出

git reset HEAD FILE_NAME

取消添加到stage区域的文件

# 将a.txt 从working directory 添加到 staging area
git add a.txt

# 将a.txt从staging area中去除
git reset HEAD a.txt

将本地仓库提交到远程仓库

主要用到这两个命令：

  1. git remote add origin githut_project.git
  2. git push -u orgin master

如：

git init

git add .

git commit -m "Initinal Version Project"

# 在github上新建一个仓库
# 比如 https://github.com/JamesSwayer/FirstProject.git

# 连接到远程仓库
git remote add origin https://github.com/JamesSwayer/FirstProject.git

# 提交到远程仓库
git push -u origin master
# 会出现要求输入github账号

# 将分支提交到github
git push -u origin new_branch

# 将所有的分支都提交
git push --all origin

注意事项：

  • 如果一开始将仓库名写错了，比如：”git remote add origin https://....first.git“, 可以使用 git remote rm origin 来移除该origin

将远程仓库同步到本地仓库

如果仓库做出变化，本地希望同步，可以使用下面命令：

  • git remote -v: 查看远程仓库地址
  • git fetch origin： 仅仅取回到本地仓库，但是变化部分不取回
  • git pull remote_git_repository

比如：

git remote -v
$ origin https://github.com/JamesSwayer/FirstProject.git (fetch)
$ origin https://github.com/JamesSwayer/FirstProject.git (push)

# 取回提交时的项目到本地
git fetch origin

# 取回远程仓库变化之后的项目到本地
git pull origin https://github.com/JamesSwayer/FirstProject.git

如果本地仓库新添加了文件，要同步到远程仓库

git add newfile.txt

git commit -m "add a new file"

# 添加到远程仓库
git pull

如果要更改远程仓库的名字，默认为 origin

使用 git remote rename originname newNAME

git remote -v
$ origin https://github.com/JamesSwayer/FirstProject.git (fetch)
$ origin https://github.com/JamesSwayer/FirstProject.git (push)

# 改名
git remote rename origin sf

git remote -v
$ sf https://github.com/JamesSwayer/FirstProject.git (fetch)
$ sf https://github.com/JamesSwayer/FirstProject.git (push)

分支操作

远程仓库操作

1- git push origin branch_name

将分支同步到远程仓库中

git push origin demo_branch

2- git checkout -b branch_name origin/branch_name

将远程仓库中的分支取回到本地

git checkout -b demo_branch origin/demo_branch

查看操作

1- git branch

查看分支

git branch

*master
demo_branch

2- git branch --merged | git branch --no-merged

查看合并了的或未合并的分支

git branch --merged

$ *master

git branch --no-merged

$ demo_branch

3- git branch -v

查看分支情况以及最后一次的commit

git branch -v

$   demo   8297267 save gitignore & webpack.config.js
$ * master d3f66de Create README.md

4- git merge branch_name

合并分支

git merge demo_branch

5- git branch -d branch_name

删除合并了的分支，注意和下面的差异

git branch demo_branch

git branch

$ * master

6- git branch -D branch_name

删除未合并的分支

7- git push origin :branch_name

删除远程仓库中的分支

git push origin :demo_branch
